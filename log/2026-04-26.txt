## Changes Made

### Change 1: Add Stage 1.2b for Source-of-Truth Parsing
- **File(s)**: `workflows/create-project-rules.md`
- **What**: Added sub-stage §1.2b to extract design tokens from source files instead of READMEs.
- **Why**: The workflow generated incorrect color and font values based on design intent rather than actual configuration.
- **Impact**: AI agents will now parse config files (like `tailwind.config.ts`) to ensure accuracy.

### Change 2: Enhance Directory and Dependency Scan
- **File(s)**: `workflows/create-project-rules.md`
- **What**: Replaced Stage 1.3 with deeper directory scanning and added Stage 1.3b to verify constraints.
- **Why**: Directory mapping missed important routes and dependencies, and constraints like "pure functions" were hallucinated.
- **Impact**: Ensures complete architectural mapping and prevents invention of constraints.

### Change 3: Add Pre-Write Accuracy Gate
- **File(s)**: `workflows/create-project-rules.md`
- **What**: Added Stage 4.5 to verify claims before writing final output files.
- **Why**: Post-hoc verification caught errors too late.
- **Impact**: Halts generation if values differ from actual codebase contents, improving accuracy.

### Change 4: Add Runtime Extraction Function
- **File(s)**: `scripts/project_rules_runtime.py`
- **What**: Added `extract_design_tokens` function.
- **Why**: To provide programmatic verification capability for extracting design tokens.
- **Impact**: Allows scripts to programmatically parse CSS/Tailwind configs.

## Decisions Made

### Decision 1: Enhancing Workflow Instructions vs. Pure Runtime Code
- **Context**: The accuracy issues could be fixed either by adding Python verification logic or explicitly telling the LLM to verify them.
- **Options Considered**: Add complex parsing to `project_rules_runtime.py` or harden the Markdown workflow.
- **Chosen**: Hardened the Markdown workflow and added a basic `extract_design_tokens` hook.
- **Rationale**: Workflow instructions keep the tool framework-agnostic while still guiding the LLM away from hallucinations.
